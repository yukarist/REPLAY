# 🛠️ REPLAY - For Streamers

### 配信特化型リアルタイムログ支援ツール

配信中の会話や雑談をリアルタイムで記録し、
あとから振り返りや切り抜き確認をしやすくするために制作した、
配信者向けのサポートアプリです。

---

# ✨ 主な機能

* 🎤 リアルタイム音声認識
* 🤖 AI文章補正（Gemini / Ollama 切替対応）
* 📝 AI配信要約
* ⭐ お気に入りログ管理
* 📅 日付別ログ表示
* 📁 CSV出力 / 要約テキスト出力
* 🌊 リアルタイム波形アニメーション
* 🔇 ノイズ軽減 / フィラー抑制
* 🎛 マイク感度調整

---

# 🔰 基本の使い方（Quick Start）

一番シンプルな使い方はこれだけです。

1. アプリを起動
2. 使用するマイクを選択
3. 配信開始時に `Start`
4. 配信終了時に `Stop`

停止時には、その日のログ要約が自動生成されます。

---

# 🤖 AIモードについて

REPLAYでは、用途に合わせて
2種類のAIモードを切り替えて利用できます。

---

## 🟥 Gemini（推奨）

Google Gemini API を利用したクラウドAIです。

高精度な文章補正・要約が可能で、
PCへの負荷も少ないため、一番おすすめのモードです。

### 必要なもの

* Gemini API Key

### 設定方法

1. 設定画面を開く
2. APIキーを入力
3. AIモードで `高精度（遅延あり）` を選択

---

### ⚠ Gemini API 制限について

無料版の Gemini API には、

```text id="r3z4ht"
1分間あたりの利用回数制限
```

があります。

そのため、長時間の連続トークやマシントーク時には、
リアルタイム補正が一時的に制限される場合があります。

※ 配信終了後の「AI要約」や「CSV / テキスト出力」にはほぼ影響ありません。

---

### 💡 快適に使うコツ

* リアルタイム補正重視 → Ollama
* 高精度な補正・要約重視 → Gemini

という使い分けがおすすめです。

---

## 🟦 Ollama（ローカルAI）

お使いのPC内で動作するローカルAIです。

インターネット通信なしで利用でき、
ログデータを完全ローカル環境で処理できます。

### 必要なもの

* Ollama
* AIモデル

推奨モデル：

```bash id="7n5x3o"
ollama pull qwen2.5:7b
ollama pull gemma2:9b
```

### 設定方法

1. Ollama を起動
2. REPLAY を起動
3. AIモードで `軽い（高速）` を選択

---

# 🔒 セキュリティについて

## ログデータはローカル保存

認識したログやお気に入りデータは、
お使いのPC内（`speech.db`）に保存されます。

外部サーバー等へ勝手に送信されることはありません。

※ 旧バージョンの `/logs` フォルダが存在する場合、
初回起動時に自動移行されます。

---

## 外部送信はAI利用時のみ

* Gemini使用時
  → 文章補正・要約のため Google Gemini API を利用

* Ollama使用時
  → 完全ローカル環境内のみで処理

---

# ⚠ 注意事項

* Windows専用アプリです
* 初回起動時、マイク許可が必要な場合があります
* APIキーは他人へ共有しないでください
